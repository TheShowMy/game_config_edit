# 游戏配置编辑工具 — 需求设计文档

> 版本: v1.0 | 技术栈: Rust + Dioxus Desktop | 风格: 简约护眼

---

## 1. 项目定位

面向游戏策划/开发的 CSV 配置表编辑工具，核心思路是 **VS Code 的导航体验 + Excel 的表格视图**，专注 `.csv` 配置文件的浏览、搜索和编辑。

---

## 2. 技术选型

| 层 | 选型 | 说明 |
|---|---|---|
| UI 框架 | Dioxus 0.7 (desktop) | wry WebView 渲染，打包体积 <5MB |
| CSV | `csv` crate | 成熟的 CSV 读写 |
| 文件遍历 | `walkdir` | 递归目录扫描 |
| CLI 参数 | `clap` (derive) | 命令行解析 |
| 文件对话框 | `rfd` | 原生文件夹选择弹窗 |
| 异步 | `tokio` | 大量文件 IO 不阻塞 UI |
| 样式 | 内联 CSS | 简约暗色 Nord 风 |

---

## 3. 启动方式

| 模式 | 触发 | 行为 |
|---|---|---|
| GUI | 双击 .app/.exe | 弹出原生文件夹选择对话框 → 进入主界面 |
| CLI (历史) | `gconf` | 打开上次退出时的工作目录 |
| CLI (当前) | `gconf .` | 以当前工作目录启动 |
| CLI (指定) | `gconf /path/to/configs` | 以指定路径启动 |

## 4. 主界面布局

采用 **左-右** 分栏布局，左边为文件列表/目录树，右边为编辑器区域。

```
┌────────────────────┬─────────────────────────────────────────────┐
│  🔍 搜索配置...    │  ▣ hero_basic.csv  ▣ hero_skill.csv     ×   │ ← Tab 栏
│────────────────────│─────────────────────────────────────────────│
│                    │  说明行 │ id  │ name     │ hp  │ atk │ ...  │ ← 表头(说明)
│  📄 hero_basic.csv │  字段名 │ id  │ name     │ hp  │ atk │ ...  │ ← 表头(字段)
│    configs/hero/   │  类型行 │ num │ string   │ num │ num │ ...  │ ← 类型行(浅色斜体)
│         120 × 15   │ ────────┼─────┼──────────┼─────┼─────┼────  │
│                    │       1 │ 101 │ Arthur   │ 500 │ 60  │ ...  │
│  📄 hero_skill.csv │       2 │ 102 │ Merlin   │ 300 │ 90  │ ...  │ ← 数据行
│    configs/hero/   │       3 │ 103 │ Lancelot │ 450 │ 75  │ ...  │    (行号从
│         350 × 8    │       4 │ ... │ ...                          │     数据起始)
│                    │                                             │
├────────────────────┴─────────────────────────────────────────────┤
│  hero_basic.csv  │ 共 120 行 · 15 列 │ UTF-8  │  Ln 3, Col 1    │ ← 状态栏
└──────────────────────────────────────────────────────────────────┘
       ↑ 左边栏                              ↑ 右边编辑区
   (宽度 ~280px,                         (剩余空间)
    可拖动调整)
```

## 5. 核心功能详述

### 5.1 左边栏 — 配置列表

- **默认平铺列表**: 直接展示所有 `.csv` 文件，文件名大号主文字，下方小号浅色字显示相对路径
- **可切换文件树模式**: 通过按钮切换为目录树结构，按文件系统目录层级递归展示，支持展开/折叠
- **文件信息**: 每个文件项右上角以浅色小字显示 `行数 × 列数`
- **搜索过滤**: 侧边栏顶部输入框，实时过滤文件名/路径
- **未保存标记**: 文件修改但未保存时，文件名前显示 `●` 黄色圆点
- **功能:**
  - 单击 → 以只读模式在右侧预览（不创建 Tab）
  - 双击 → 打开到右侧编辑区（创建 Tab）
  - 右键菜单预留（刷新、重命名、在资源管理器打开）

**列表模式 vs 树模式示意:**

```
  列表模式 (默认):                    树模式 (可切换):
  ┌──────────────────────┐          ┌──────────────────────┐
  │ 📄 hero_basic.csv    │          │ 📁 configs/          │
  │   configs/hero/      │          │   📁 hero/           │
  │    120 × 15          │          │     📄 hero_basic   │
  │                      │          │     📄 hero_skill   │
  │ 📄 hero_skill.csv    │          │   📁 item/           │
  │   configs/hero/      │          │     📄 item_template│
  │    350 × 8           │          │   📁 level/          │
  │                      │          │     📄 stage_config │
  │ 📄 item_template.csv │          └──────────────────────┘
  │   configs/item/      │
  │    2000 × 20         │
  │                      │
  │ 📄 stage_config.csv  │
  │   configs/level/     │
  │    50 × 10           │
```

### 5.2 右边栏 — 编辑器

**两种视图可切换:**

| 模式 | 说明 | 适用场景 |
|---|---|---|
| **表格视图** (默认) | 类似 Excel，列表头固定、行号固定列、虚拟滚动 | 浏览数据 |
| **文本视图** | 原始 CSV 文本编辑，等宽字体 | 快速改原始文本 |

**表格视图的三种交互状态:**

| 状态 | 进入方式 | 退出方式 | 说明 |
|---|---|---|---|
| **导航态** (默认) | 打开文件即进入 | — | 键盘方向键浏览单元格，`T` 进入聚焦模式 |
| **聚焦列模式** | 导航态中按 `T` | `T` 或 `Esc` | 选中列展开，左右邻居列保留 |
| **编辑态** | 导航态或聚焦模式中单击单元格 | `Esc` | 直接修改单元格内容 |

> `Esc` 行为：编辑态 → 退回聚焦模式或导航态；聚焦模式 → 退回导航态；导航态 → 关闭弹窗/面板。

---

#### 表格视图 — 导航态

- 左侧固定行号列（# 列），从数据行开始计数
- 单元格选中高亮
- 列宽固定，可拖拽分隔线调整
- 虚拟滚动，大数据量不卡
- 方向键移动选中单元格

#### 表格视图 — 表头配置

- 表头行数可设置，默认 2 行
- 第 1 行: 说明/注释行，以浅色小号字显示在表头区域
- 第 2 行: 字段名，作为列标识
- 设置入口：菜单/工具栏 → 表头行数设置（1~5 行可选）

#### 表格视图 — 类型行（仅展示，不写入 CSV）

在表头和数据之间插入一行，自动推断每列的数据类型。

根据该列所有数据行的值判断类型:

| 类型 | 示例 | 判断规则 |
|---|---|---|
| `string` | `hello` | 纯文本 |
| `number` | `123`、`3.14` | 可解析为整数或浮点数 |
| `bool` | `true`、`false`、`0`、`1` | 布尔值 |
| `json` | `{"key": "value"}` | JSON 对象 |
| `array` | `[1, 2, 3]` | JSON 数组 |
| `array_2d` | `[[1,2],[3,4]]` | JSON 二维数组 |

- **数组的两种表示形式均需识别:**
  - JSON 数组格式: `["a","b"]` / `[["a","b"],["c","d"]]`
  - 分隔符格式: `"a","b"` / `"a","b";"c","d"`（引号内为元素，逗号分隔一维，分号分隔二维）
- **类型推断优先级（从高到低）:** `bool` → `number` → `array_2d` → `array` → `json` → `string`
- **若所有行均为空 → 显示 `string`**
- 类型行以浅色斜体展示，视觉上与表头和数据行明显区分

#### 表格视图 — 警告规则

配置可能被转换为 JSON 格式使用，所有会导致 JSON 解析/转换异常的值都需要警告。

| 场景 | 级别 | 处理 |
|---|---|---|
| 数字 + 文本混合 | ✅ 无警告 | 列类型归为 `string` |
| `number` 与 `bool` 混合 | ⚠️ 黄色 `mixed` | 类型行该列黄色背景警告 |
| `json` 与文本混合 | ⚠️ 黄色 `mixed` | 类型行该列黄色背景警告 |
| 其他异构混合 | ⚠️ 黄色 `mixed` | 类型行该列黄色背景警告 |
| JSON 格式错误 | 🔴 红色 | 类型行该列显示红色 `json*`，具体出错单元格红色标记 |
| 任意字段值中含真实的换行控制字符 | 🔴 红色 | 类型行该列红色标记，具体单元格红色标记 |
| 任意字段值中含不可见字符（tab、全角空格、零宽字符等） | 🔴 红色 | 类型行该列红色标记，具体单元格红色标记 |
| 任意字段值中含未转义的特殊 JSON 字符 | 🔴 红色 | 同上 |

> - 红色项适用于**所有列**，不论该列被推断为何种类型。因为配置终将转 JSON，任何会导致 JSON 序列化失败的值都要标记。
> - 正常的文本转义 `\n`（反斜杠+n）是允许的，**不**触发警告。仅真实的换行控制字符（ASCII 0x0A/0x0D）需要警告。

**错误导航:**
- 状态栏显示当前文件的错误总数（如 `🔴 3 errors`，黄/红分开计数）
- `F8` → 跳转到下一个错误位置，自动滚动并选中对应单元格
- `Shift+F8` → 跳转到上一个错误位置
- 点击类型行中的红色标记直接跳转到该列第一个错误单元格

#### 表格视图 — 聚焦列模式

方便查看长文本和复杂 JSON 内容。

- **触发**: 导航态中选中某列，按 `T` 进入；再次按 `T` 或 `Esc` 退出，退回导航态
- **列宽**: 选中列自动扩宽以适应该列内容（非强制最大，根据实际内容合理撑开），左右各保留一列邻居列（前一列、后一列），其余列隐藏
- **列切换**: `A` / `←` 聚焦前一列，`D` / `→` 聚焦后一列；也可以直接鼠标点击左右邻居列的任意单元格，切换聚焦到该列
- **行高**: 单元格内容多行展示，行高有最大限制（如 6 行），超出截断
- **编辑**: 聚焦模式中单击单元格可进入编辑态，编辑时该单元格出现滚动条

#### 表格视图 — 单元格编辑

- **进入**: 导航态或聚焦列模式中，单击单元格进入编辑态
- **编辑态**: 单元格内容超出展示限制时出现滚动条，可滚动查看完整内容
- **退出**: `Esc` 退出编辑态，退回原状态（聚焦模式或导航态），内容恢复截断展示
- **非编辑态**: 超出部分直接截断，不显示滚动条

**JSON 单元格特殊处理:**
- **展示时**: JSON 内容自动格式化缩进展示（pretty print），方便阅读
- **保存时**: 格式化带来的缩进和换行会被去掉，恢复为紧凑格式写入 CSV，不影响原始数据
- **编辑时**: 提供基本的 JSON 语法高亮（键名、字符串、数字、布尔、括号等用不同颜色区分）

---

#### 文本视图

- 等宽字体、纯文本展示
- 行号侧边栏
- 基础编辑能力（增删改）
- 此模式下无表格视图的概念（无表头行、类型行、聚焦列等），就是原始 CSV 文本

### 5.3 Tab 管理

- 双击文件或打开新文件 → 新建 Tab
- Tab 显示文件名，带 `×` 关闭按钮
- 未保存 Tab 显示圆点标记
- `Ctrl+Tab` / `Ctrl+Shift+Tab` 切换 Tab
- `Ctrl+W` 关闭当前 Tab
- 最多同时打开 20 个 Tab（超出提示关闭旧 Tab）

### 5.4 命令面板 (Ctrl+P)

- **触发**: `Ctrl+P`，全屏半透明遮罩 + 居中搜索框
- **搜索匹配优先级**: 文件名 → 目录路径 → 文件内容
- **模糊匹配**: 输入 `hero_basic` 或 `hbasic` 或 `basic` 都能查到
- **行号跳转**: 输入 `:120` 回车 → 直接跳转到当前打开文件的第 120 行（此行为等价于 VS Code）
- **结果列表**: 每项显示文件图标 + 文件名 + 路径摘要
- **键盘操作**: `↑↓` 选择，`Enter` 打开，`Esc` 关闭

```
┌──────────────────────────────────────┐
│                                       │
│   > hbasic                            │  ← 输入框
│  ──────────────────────────────────   │
│   📄 hero_basic.csv                   │
│      configs/hero/hero_basic.csv      │
│   📄 hero_basic_test.csv              │  ← 结果列表
│      configs/test/hero_basic_test.csv │
│                                       │
└──────────────────────────────────────┘
```

### 5.5 行号跳转 (Ctrl+G)

- 按 `Ctrl+G` 弹出小型行号输入框
- 输入数字回车 → 编辑器滚动到对应行并高亮 2 秒
- 同样支持通过 `Ctrl+P` + `:行号` 实现

### 5.6 文件保存

- `Ctrl+S` 保存当前激活 Tab
- 保存时检测编码（保持原编码或统一 UTF-8 BOM）
- 保存成功后状态栏提示 "已保存"
- 关闭未保存 Tab 时弹窗确认（保存/不保存/取消）

## 7. 视觉风格

- **暗色主题**: 深色背景、柔和文字，长时间使用不刺眼
- **简约克制**: 少用彩色，功能区域靠灰度层次区分，不追求花哨装饰
- **清晰层级**: 侧边栏、编辑区、状态栏通过背景深浅自然分隔
- **柔和反馈**: 悬停高亮、选中态、未保存标记等用低饱和度颜色，不突兀
- **等宽优先**: 编辑区和数据展示区使用等宽字体 (Cascadia Code / Fira Code / 系统默认)
