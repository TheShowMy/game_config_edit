# 游戏配置编辑工具 — 需求设计文档

> 版本: v1.0 | 技术栈: Rust + Dioxus Desktop | 风格: 简约护眼

---

## 1. 项目定位

面向游戏策划/开发的 CSV 配置表编辑工具，核心思路是 **VS Code 的导航体验 + Excel 的表格视图**，专注 `.csv` 配置文件的浏览、搜索和编辑。

---

## 2. 技术选型

| 层 | 选型 | 说明 |
|---|---|---|
| UI 框架 | Dioxus 0.7 (desktop) | wry WebView 渲染，使用系统 WebView 运行时 |
| CSV | `csv` crate | 成熟的 CSV 读写 |
| 文件遍历 | `walkdir` | 递归目录扫描 |
| CLI 参数 | `clap` (derive) | 命令行解析 |
| 文件对话框 | `rfd` | 原生文件夹选择弹窗 |
| 异步 | `tokio` | 大量文件 IO 不阻塞 UI |
| 样式 | 内联 CSS | 简约暗色 Nord 风 |

v1 支持并验收以下两个平台：

- Windows 10/11 x64（`win32-x64`）
- macOS Apple Silicon（`darwin-arm64`，M1 及后续芯片）

Windows ARM64、macOS Intel 和 Linux 不在 v1 支持范围内，必须显示明确的“不支持当前平台”错误，不能回退使用其他平台二进制。

## 3. 启动方式

| 模式 | 触发 | 行为 |
|---|---|---|
| GUI | 双击 Windows `gconf.exe` / macOS `gconf` 可执行文件 | 与无参数 CLI 行为一致 |
| CLI (历史) | `gconf` | 打开上次退出时的有效工作目录；无历史目录或目录已失效时弹出文件夹选择对话框 |
| CLI (当前) | `gconf .` | 以当前工作目录启动 |
| CLI (指定) | `gconf /path/to/configs` | 以指定路径启动 |

- 显式传入的路径不存在、不可读或不是目录时，显示错误并退出，不回退到其他目录。
- 主界面提供“打开文件夹”入口。切换工作目录前按未保存文件退出规则处理。
- 最近工作目录保存在用户级应用配置目录中，成功打开目录后更新。

### 3.1 文件格式约定

- 只接受 UTF-8 CSV；允许文件带 UTF-8 BOM。读取失败时以只读错误页显示非法字节位置，不做替换解码。
- 保存时保持原文件是否带 BOM，并保持原有 `LF` / `CRLF` 换行风格；新文件默认使用无 BOM UTF-8 和 `LF`。
- 默认分隔符为逗号。打开文件时在逗号、Tab、分号、竖线四种候选中自动识别；无法得到唯一可靠结果时使用逗号并在状态栏提示。
- 自动识别以“能合法解析且前 50 条非空记录的列数最稳定”为准；只有单列数据时不猜测其他分隔符。
- 状态栏/工具栏允许为当前文件手动选择逗号、Tab、分号或竖线。切换后立即重新解析，选择结果在当前工作目录内按相对路径记忆；选择分隔符本身不修改 CSV。
- 引号和双引号转义遵循 RFC 4180。允许引号字段包含分隔符和换行；列数不一致、引号未闭合等解析错误必须显示位置，并禁止进入表格视图。
- 保存使用当前文件选定的分隔符。未发生表格结构编辑时保留原始文本；发生表格编辑后允许规范化受影响记录的引号写法，但不得改变解析后的字段值。

## 5. 核心功能详述

### 5.1 左边栏 — 配置列表

- **默认平铺列表**: 直接展示所有 `.csv` 文件，文件名大号主文字，下方小号浅色字显示相对路径
- **可切换文件树模式**: 通过按钮切换为目录树结构，按文件系统目录层级递归展示，支持展开/折叠
- **文件信息**: 每个文件项右上角以浅色小字显示 `行数 × 列数`
- **搜索过滤**: 侧边栏顶部输入框，实时过滤文件名/路径
- **未保存标记**: 文件修改但未保存时，文件名前显示 `●` 黄色圆点
- **功能:**
  - 单击 → 以只读模式在右侧预览（不创建 Tab）
  - 双击 → 打开到右侧编辑区（创建 Tab）
  - v1 不实现右键菜单、重命名和删除文件；“在系统文件管理器中显示”通过工具栏提供（Windows Explorer / macOS Finder）

- 扫描扩展名时不区分大小写，只递归当前工作目录且不跟随目录符号链接；单个目录不可读时保留其他结果并显示扫描警告。
- 文件信息中的行数指扣除已配置表头后的数据记录数，列数指合法记录的字段数；解析失败时显示错误标记而不是可能错误的行列数。
- 行列统计在后台执行，先显示文件列表，再逐步补充统计结果，不得因统计全部文件而阻塞首屏。

**预览规则:**

- 同一时间最多存在一个临时预览；再次单击其他文件会替换它。
- 预览期间显示“只读预览”标记，不允许进入编辑态，`Ctrl+S` 不执行保存。
- 预览表格中的单元格可以单击选择，也可使用 `T` 进入/退出聚焦列模式；聚焦后支持 `A` / `D`、左右方向键和点击邻列切换聚焦，但任何操作都不得创建编辑草稿或修改文件。
- 双击当前预览文件会将其打开为普通 Tab；如果该文件已有 Tab，则激活已有 Tab，不重复打开。
- 点击任意 Tab 会关闭临时预览并恢复该 Tab。预览时按 `Ctrl+W` 只关闭预览并返回之前的 Tab。
- `Ctrl+F`、行号跳转和聚焦列导航可用于预览；保存、粘贴、撤销、重做等修改命令仍只对正式 Tab 开放。

**列表模式 vs 树模式示意:**

```
  列表模式 (默认):                    树模式 (可切换):
  ┌──────────────────────┐          ┌──────────────────────┐
  │ 📄 hero_basic.csv    │          │ 📁 configs/          │
  │   configs/hero/      │          │   📁 hero/           │
  │    120 × 15          │          │     📄 hero_basic   │
  │                      │          │     📄 hero_skill   │
  │ 📄 hero_skill.csv    │          │   📁 item/           │
  │   configs/hero/      │          │     📄 item_template│
  │    350 × 8           │          │   📁 level/          │
  │                      │          │     📄 stage_config │
  │ 📄 item_template.csv │          └──────────────────────┘
  │   configs/item/      │
  │    2000 × 20         │
  │                      │
  │ 📄 stage_config.csv  │
  │   configs/level/     │
  │    50 × 10           │
  └──────────────────────┘
```

### 5.2 右边栏 — 编辑器

**两种视图可切换:**

| 模式 | 说明 | 适用场景 |
|---|---|---|
| **表格视图** (默认) | 类似 Excel，列表头固定、行号固定列、虚拟滚动 | 浏览数据 |
| **文本视图** | 原始 CSV 文本编辑，等宽字体 | 快速改原始文本 |

**双视图数据同步:**

- 每个已打开文件维护一份 UTF-8 原始文本和最近一次成功解析的表格快照。
- 文本视图编辑后切换到表格视图时执行完整解析；解析失败则留在文本视图，显示第一个错误及错误总数。
- 表格视图编辑的是解析后的字段值；切换到文本视图时按当前方言更新原始文本缓冲区。
- 仅浏览或切换视图不会标记未保存，也不会重写原始文本。只有表格数据实际修改后才允许重新序列化受影响记录。
- 表格快照不是独立副本，两个视图共享同一份未保存状态，保存和撤销均作用于当前文件。

**表格视图的三种交互状态:**

| 状态 | 进入方式 | 退出方式 | 说明 |
|---|---|---|---|
| **导航态** (默认) | 打开文件即进入 | — | 键盘方向键浏览单元格，`T` 进入聚焦模式 |
| **聚焦列模式** | 导航态中按 `T` | `T` 或 `Esc` | 选中列展开，左右邻居列保留 |
| **编辑态** | 单击已选单元格，或按 `Enter` / `F2` | `Enter` 提交；`Esc` 取消 | 直接修改单元格内容 |

> `Esc` 行为：编辑态取消本次未提交输入并退回聚焦模式或导航态；聚焦模式退回导航态；导航态关闭最上层弹窗/面板。

#### 表格视图 — 导航态

- 左侧固定行号列（# 列），从数据行开始计数
- 单元格选中高亮
- 列宽固定，可拖拽分隔线调整
- 使用虚拟滚动，具体性能按第 8 节基线验收
- 方向键移动选中单元格

#### 表格视图 — 表头配置

- 表头行数可设置，默认 2 行
- 设置为 `N` 行时，第 `N` 行为字段名，前 `N-1` 行均为说明/注释行，数据从第 `N+1` 条 CSV 记录开始
- 设置为 1 行时只有字段名，没有说明行
- 多个说明行按原顺序显示在字段名上方，以浅色小号字展示
- 设置入口：菜单/工具栏 → 表头行数设置（1~5 行可选）
- 表头行数是逐文件配置，在当前工作目录内按相对路径持久化；新文件继承工作目录默认值 2
- 修改表头行数只改变工作目录配置，不修改 CSV，也不标记文件为未保存
- 文件记录数少于表头行数时显示配置错误，不推断类型，也不进入可编辑表格

#### 表格视图 — 类型行（仅展示，不写入 CSV）

在表头和数据之间插入一行，扫描该列所有数据记录后推断整列类型。空单元格不参与类型判断，也不单独产生类型错误。后台扫描完成前类型行显示“分析中”，不展示可能变化的临时结论。

单元格基础类型:

| 类型 | 示例 | 判断规则 |
|---|---|---|
| `string` | `hello` | 纯文本 |
| `number` | `123`、`-3.14`、`1e3` | 符合 JSON number 语法的有限数值 |
| `bool` | `true`、`false`、`0`、`1` | 大小写敏感的布尔值 |
| `json` | `{"key": "value"}` | 可完整解析的 JSON 对象 |
| `array` | `[1, 2, 3]` | JSON 一维数组或合法的一维分隔符数组 |
| `array_2d` | `[[1,2],[3,4]]` | JSON 二维数组或合法的二维分隔符数组 |

- **数组的两种表示形式均需识别:**
  - JSON 数组格式: `["a","b"]` / `[["a","b"],["c","d"]]`
  - 分隔符格式指 **CSV 解析后的单个字段值** 为 `"a","b"` / `"a","b";"c","d"`，不是原始 CSV 中相邻的多个字段
  - 分隔符数组的每个元素必须被双引号完整包围，元素内的双引号用 `""` 表示；逗号分隔一维元素，分号分隔二维分组
- **整列推断规则:**
  - 先忽略空值并扫描整列；若所有值均为空，类型为 `string`
  - 只要存在合法 JSON 对象，该列按 `json` 检查；其他非空值必须也是合法 JSON 对象，否则对应单元格标红，类型行显示 `json*`
  - 不存在 JSON 对象但存在合法二维数组时，该列按 `array_2d` 检查；不存在前两者但存在合法一维数组时，按 `array` 检查；不符合该列结构类型的非空值标红
  - 不存在结构化值时：全部为 bool 则为 `bool`，全部为 number 则为 `number`，bool 与 number 混合为黄色 `mixed`
  - 数字与普通文本混合归为 `string` 且不警告；其他基础类型混合为黄色 `mixed`
  - 仅以 `{` 或 `[` 开头但无法解析的文本，在同列没有任何合法结构化值时按普通 `string` 处理；一旦同列存在合法结构化值，则将其标为结构错误
- 类型行以浅色斜体展示，视觉上与表头和数据行明显区分

#### 表格视图 — 警告规则

部分下游转换工具会直接拼接字段值而不调用标准 JSON 序列化器，因此本工具除类型错误外，还检查可能破坏这类转换流程的危险字符。此类标记表示兼容性风险，不表示标准 JSON 序列化一定失败。

| 场景 | 级别 | 处理 |
|---|---|---|
| 数字 + 文本混合 | ✅ 无警告 | 列类型归为 `string` |
| `number` 与 `bool` 混合 | ⚠️ 黄色 `mixed` | 类型行该列黄色背景警告 |
| 其他非结构化基础类型混合 | ⚠️ 黄色 `mixed` | 类型行该列黄色背景警告 |
| 已判定为结构化列但单元格格式不匹配 | 🔴 红色 | 类型行显示 `json*` / `array*` / `array_2d*`，具体单元格红色标记 |
| 任意字段值中含真实的换行控制字符 | 🔴 红色 | 类型行该列红色标记，具体单元格红色标记 |
| 任意字段值中含危险的不可见字符（Tab、全角空格、零宽字符等） | 🔴 红色 | 类型行该列红色标记，具体单元格红色标记 |
| 普通字符串中含未转义的 `"` 或非法反斜杠转义 | 🔴 红色 | 合法转义仅包括 `\"`、`\\`、`\/`、`\b`、`\f`、`\n`、`\r`、`\t`、`\uXXXX` |

> - 字符类红色项适用于**所有列**；结构化 JSON/数组自身语法所需的引号和反斜杠不重复按普通字符串规则告警。
> - 正常的文本转义 `\n`（反斜杠+n）是允许的，**不**触发警告。仅真实的换行控制字符（ASCII 0x0A/0x0D）需要警告。
> - 保存只负责正确执行 CSV 引号转义，不会自动添加或删除字段内部的 JSON 转义。危险内容由用户确认和修正，避免静默改变配置语义。

**错误导航:**
- 状态栏分别显示红色单元格问题数和黄色问题列数；同一单元格命中多条红色规则时只计一次
- `F8` → 按数据记录号、列号顺序跳转到下一个红色单元格；红色问题遍历完后跳转到黄色类型列，并在末尾循环
- `Shift+F8` → 跳转到上一个错误位置
- 点击类型行中的红色标记直接跳转到该列第一个错误单元格

#### 表格视图 — 聚焦列模式

方便查看长文本和复杂 JSON 内容。

- **触发**: 导航态中选中某列，按 `T` 进入；再次按 `T` 或 `Esc` 退出，退回导航态
- **可视区域**: 通过表格滚动区域的真实宽高计算布局，窗口缩放、DPI、侧栏隐藏或拖动改变宽度时立即重算。
- **聚焦列**: 聚焦列按内容在 `320–720px` 内取宽，空间不足时缩小；将 58px 行号栏计入布局后，聚焦列视觉中心必须与表格可视区域中心重合。
- **邻近列**: 左右两侧分别以 180px 为基准，从聚焦列向外依次显示能容纳的所有邻近列，数量不设上限，只受该侧实际空间和剩余列数限制。
- **单侧不足**: 某侧实际列数少于容量时，已有列均分并铺满该侧；该侧没有源列时保留等宽空白，不把空间借给另一侧，聚焦列始终居中。
- **列切换**: `A` / `←` 聚焦前一列，`D` / `→` 聚焦后一列；也可以直接鼠标点击左右邻居列的任意单元格，切换聚焦到该列
- **行高**: 单元格内容多行展示，行高有最大限制（如 6 行），超出截断
- **编辑**: 聚焦模式中单击单元格可进入编辑态，编辑时该单元格出现滚动条

#### 表格视图 — 单元格编辑

- **进入**: 导航态或聚焦列模式中，单击已选单元格或按 `Enter` / `F2` 进入编辑态；单击未选单元格只移动选择
- **编辑态**: 单元格内容超出展示限制时出现滚动条，可滚动查看完整内容
- **提交**: 普通单元格中，`Enter` 提交并移动到下一数据行，`Tab` / `Shift+Tab` 提交并移动到相邻列；点击其他单元格时先提交再移动
- **取消**: `Esc` 丢弃本次尚未提交的输入，退回原状态（聚焦模式或导航态）
- **非编辑态**: 超出部分直接截断，不显示滚动条
- **剪贴板**: `Ctrl+C` 复制当前单元格原始值，`Ctrl+V` 覆盖当前单元格；v1 不支持矩形多单元格选择和批量粘贴
- **撤销**: 每个 Tab 独立维护编辑历史，`Ctrl+Z` 撤销、`Ctrl+Y` 重做，历史保留到 Tab 关闭
- v1 表格视图只修改已有单元格，不提供新增/删除行列；结构调整通过文本视图完成

**JSON 单元格特殊处理:**
- **展示时**: JSON 内容在渲染层自动格式化缩进（pretty print），不修改底层字段值，也不因此产生真实换行告警
- **编辑时**: 编辑器可以临时展示格式化文本，并提供基本的 JSON 语法高亮（键名、字符串、数字、布尔、括号等用不同颜色区分）；`Enter` 插入格式化换行，`Tab` 缩进，`Ctrl+Enter` 提交
- **提交时**: 必须能解析为相同结构类型；成功后压缩为紧凑 JSON 写回字段值，失败则保留编辑态并显示语法错误；`Esc` 取消时恢复提交前的原始值
- 未编辑的 JSON 单元格在保存时保持原字段值，不因展示格式化而被改写

#### 文本视图

- 等宽字体、纯文本展示
- 行号侧边栏
- 基础编辑能力（增删改）
- 使用系统文本编辑惯例支持选择、剪切、复制、粘贴、`Ctrl+Z` 撤销和 `Ctrl+Y` 重做
- 此模式下无表格视图的概念（无表头行、类型行、聚焦列等），就是原始 CSV 文本

### 5.3 Tab 管理

- 双击文件或打开新文件 → 新建 Tab
- Tab 显示文件名，带 `×` 关闭按钮
- 未保存 Tab 显示圆点标记
- `Ctrl+Tab` / `Ctrl+Shift+Tab` 切换 Tab
- `Ctrl+W` 关闭当前 Tab
- 同一路径最多打开一个 Tab；从列表、搜索结果或预览重复打开时直接激活已有 Tab
- 最多同时打开 20 个 Tab；达到上限时拒绝继续打开并提示用户先关闭现有 Tab，不自动关闭任何 Tab
- 点击 Tab 关闭按钮或按 `Ctrl+W` 时，如存在未保存修改，显示“保存 / 不保存 / 取消”确认

### 5.4 命令面板 (Ctrl+P)

- **触发**: `Ctrl+P`，全屏半透明遮罩 + 居中搜索框
- **搜索范围**: 文件名和相对路径；文件内容搜索由 `Ctrl+Shift+F` 提供
- **搜索匹配优先级**: 文件名完整匹配 → 文件名前缀/模糊匹配 → 目录路径匹配
- **模糊匹配**: 输入 `hero_basic` 或 `hbasic` 或 `basic` 都能查到
- **行号跳转**: 输入 `:120` 回车 → 按当前视图的行号规则跳转；表格视图跳数据记录，文本视图跳物理行
- **结果列表**: 每项显示文件图标 + 文件名 + 路径摘要
- **键盘操作**: `↑↓` 选择，`Enter` 打开，`Esc` 关闭

```
┌──────────────────────────────────────┐
│                                       │
│   > hbasic                            │  ← 输入框
│  ──────────────────────────────────   │
│   📄 hero_basic.csv                   │
│      configs/hero/hero_basic.csv      │
│   📄 hero_basic_test.csv              │  ← 结果列表
│      configs/test/hero_basic_test.csv │
│                                       │
└──────────────────────────────────────┘
```

### 5.5 行号跳转 (Ctrl+G)

- 按 `Ctrl+G` 弹出小型行号输入框
- 表格视图中数字表示去掉表头后的 **数据记录号**，从 1 开始；文本视图中数字表示原始文本的 **物理行号**，从 1 开始
- 输入数字回车 → 编辑器滚动到对应记录/行并高亮 2 秒；超出范围时显示错误，不自动跳到末尾
- 同样支持通过 `Ctrl+P` + `:行号` 实现
- CSV 引号字段可以包含真实换行，因此同一数据记录可能跨越多个物理文本行；状态栏分别使用 `Row` 和 `Ln` 标识，二者不得混用

### 5.6 文件保存

- `Ctrl+S` 保存当前激活 Tab
- 文件始终按 3.1 节的 UTF-8、BOM、换行和分隔符规则保存
- 黄色警告不阻止保存；存在红色问题或文本视图 CSV 解析错误时，保存前显示问题数量并要求用户再次确认
- 打开文件时记录内容指纹；保存前如检测到磁盘文件被外部修改，显示“覆盖磁盘文件 / 重新加载 / 取消”，不得静默覆盖
- 保存时先写入同目录临时文件，写入和刷新成功后替换原文件；任一步骤失败时保留原文件和未保存状态，并显示可操作的错误信息
- 保存成功后状态栏提示 "已保存"
- 关闭未保存 Tab 时弹窗确认（保存/不保存/取消）
- 关闭窗口、切换工作目录时汇总所有未保存 Tab，允许“全部保存 / 全部不保存 / 返回编辑”；任一保存失败时停止关闭

### 5.7 文件内与全局搜索

- `Ctrl+F` 搜索当前预览或当前 Tab。文本视图按原始文本匹配；表格视图按解析后的单元格值匹配，并跳转选择对应单元格。
- `Ctrl+Shift+F` 搜索工作目录内全部 CSV 内容，结果显示文件、数据记录号或物理行号、命中摘要；点击结果打开文件并定位。
- 两种搜索默认区分纯文本，不使用正则表达式；提供大小写敏感开关。
- 全局搜索采用流式扫描，可随时取消；新查询会取消旧查询，不阻塞文件浏览和当前文件编辑。

### 5.8 界面语言与快捷键入口

- 应用启动时读取系统 locale；以 `zh` 开头的 locale 统一显示简体中文，其他语言及检测失败时显示英文，不提供手动切换或持久化语言设置。
- 中文应用标题为“游戏配置编辑器”。普通界面文本、动态计数、提示、原生对话框、tooltip、placeholder 和 ARIA 标签均随语言切换；`CSV`、`JSON`、`UTF-8`、路径及类型标识等技术内容保持原样。
- 可识别的启动、设置、CSV、打开和保存错误使用本地化前缀，底层 OS 或解析器详情附在前缀之后并保持原样。
- 分隔符在中文界面显示“逗号 / 制表符 / 分号 / 竖线”，英文界面显示“Comma / Tab / Semicolon / Pipe”。
- 顶部“打开文件夹”旁提供“快捷键”按钮，`F1` 在非文本输入状态打开同一面板；面板按“全局、文件与搜索、表格导航、编辑”分组列出实际支持的全部快捷键，`Esc` 或关闭按钮退出。
- 快捷键面板在 Windows 显示 `Ctrl`，macOS 显示 `Command`；Tab 切换始终显示两平台通用的 `Ctrl+Tab` / `Ctrl+Shift+Tab`。

### 5.9 外部文件变化

- 后台监视已扫描的 CSV 文件。新增、删除和重命名应更新侧边栏；已打开文件的变化必须单独处理。
- 未修改 Tab 对应的磁盘文件发生变化时自动重新加载，并保留当前选择位置；重新解析失败则切换到文本错误状态。
- 已修改 Tab 对应的磁盘文件发生变化时只提示冲突，不自动合并或覆盖；用户选择重新加载前必须二次确认会丢弃本地修改。

## 8. 性能与验收基线

以下指标分别在 Windows 11 x64（4 核 CPU、16 GB 内存、SSD）和 macOS Apple Silicon（M1 或以上、16 GB 内存、SSD）的 Release 构建上验收：

- 基准工作目录：2,000 个 CSV、总大小 2 GB；单文件上限 100 MB 或 500,000 条数据记录，任一先达到即视为上限。
- 启动后 1 秒内显示可交互窗口，3 秒内显示首批文件名；行列统计、类型推断和告警检查可继续在后台完成并显示进度。
- 打开 100 MB 基准文件后 3 秒内显示首屏数据；后台扫描期间，键盘和鼠标操作不得出现超过 100 ms 的 UI 线程阻塞。
- 表格只渲染可见区域及少量缓冲行；连续滚动期间不因数据总行数增加而持续增长 DOM 节点。
- 文件名/路径过滤在已扫描 2,000 个文件上 100 ms 内更新。全局内容搜索流式返回结果，取消请求在 500 ms 内停止。
- 单个后台任务失败不得导致应用退出；错误必须关联到具体文件，且其他文件仍可浏览和编辑。
- P6 使用固定测试数据记录首次显示耗时、峰值内存和发布包体积。发布包体积记录实测值，不以未经验证的 `<5MB` 作为验收条件。

## 9. npm 发布与分发

### 9.1 包结构

npm 使用“一个入口包 + 两个平台原生包”的结构，三个包版本必须完全一致：

| 包 | 内容 | 平台限制 |
|---|---|---|
| `game-config-edit` | JavaScript 启动器、`gconf` 命令和两个平台包的 `optionalDependencies` | 所有平台均可安装；运行时检查支持范围 |
| `game-config-edit-win32-x64` | Windows x64 Release 二进制 `gconf.exe` | `os: ["win32"]`、`cpu: ["x64"]` |
| `game-config-edit-darwin-arm64` | macOS Apple Silicon Release 二进制 `gconf` | `os: ["darwin"]`、`cpu: ["arm64"]` |

- `game-config-edit` 是面向用户的唯一安装入口：`npm install -g game-config-edit`；全局安装后提供 `gconf` 命令。
- 主包声明 `engines.node: ">=22"`，不支持的 Node.js 版本在安装阶段给出明确提示。
- 主包通过精确版本的 `optionalDependencies` 引用两个平台包，不能使用 `^`、`~` 或版本范围。
- 启动器根据 `process.platform` 和 `process.arch` 选择唯一对应的原生包，将所有 CLI 参数原样传递，并转发退出码和错误输出。
- 对应平台包缺失时，错误信息必须包含当前平台、期望的包名以及重新安装命令；不得在安装阶段从 npm 之外下载二进制。
- npm 安装提供 `gconf` CLI 入口，不执行需要管理员权限的安装脚本。双击启动针对构建产物中的 `gconf.exe` / `gconf` 可执行文件。
- npm 包名在正式发布前必须再次确认可用且归当前发布账号所有；如改用 npm scope，三个包必须统一迁移到同一 scope，不能只修改其中一个。

### 9.2 GitHub Actions 发布流程

- 向 GitHub 推送符合 `vX.Y.Z` 的 Git Tag（例如 `v1.2.3`）后自动触发正式发布工作流；不需要手工执行 `npm publish`。
- 工作流以 Tag 去掉前缀 `v` 后的值作为唯一发布版本，并校验 Cargo 包版本与该版本一致；不一致时在构建前失败。三个 npm 包的版本由工作流统一设置为该版本。
- 正式发布工作流只响应三段纯数字版本 Tag；分支推送、Pull Request、`v1.2`、`v1.2.3-beta.1` 等均不得触发正式 npm 发布。
- 使用 GitHub Actions 构建矩阵分别生成 Windows x64 和 macOS arm64 Release 产物。每个平台必须先运行格式检查、单元测试、Release 构建和原生启动冒烟测试。
- macOS 产物为未签名的单个 Apple Silicon 可执行文件，不构建 `.app`，也不执行签名、公证或 stapling。
- 发布顺序固定为：两个平台包 → `game-config-edit` 主包。主包只能在两个同版本平台包均发布成功后发布，避免用户安装到缺失二进制的版本。
- 使用 GitHub 已配置的 `NPM_TOKEN` Secret 完成 npm 身份验证；工作流和日志不得输出密钥内容。
- npm 发布启用 provenance，并使用公开包访问级别。正式版本发布到 `latest` dist-tag；同一版本重复执行时不得覆盖或生成不同内容。
- 发布前对三个包执行 `npm pack --dry-run`，校验包内文件白名单、平台字段、精确依赖版本、许可证和 README，禁止包含源码缓存、临时文件或密钥。
- 任一构建、测试或平台包发布步骤失败时，工作流必须终止且不得发布主包；修复后通过新版本 Tag 重新发布，不能覆盖 npm 已存在版本。

### 9.3 发布验收

- 在干净的 Windows x64 和 macOS arm64 环境分别执行 `npm install -g game-config-edit@<version>`。
- `gconf --version` 输出必须与发布版本一致，`gconf .` 必须成功启动对应平台原生应用并正确传递当前目录。
- Windows 安装中不得包含 macOS 原生包内容，macOS 安装中不得包含 Windows 原生包内容。
- 在不支持的平台运行时，启动器必须在启动原生进程前退出，并输出受支持平台列表。
- 发布工作流最终记录三个 npm 包的名称、版本、完整性摘要和 npm 页面链接，便于追溯。

## 10. 开发阶段拆分

| 阶段 | 内容 | 交付物 |
|---|---|---|
| **P1** | 项目脚手架、Dioxus 窗口、无参数/路径启动、最近目录、文件夹选择 | 按启动规则进入可运行窗口，非法路径有明确错误 |
| **P2** | 后台文件扫描、列表/目录树、懒加载行列统计、只读预览、文件变化监视 | 可浏览目录；扫描错误不阻塞其他文件 |
| **P3** | UTF-8 与方言识别、CSV 解析、表头配置、虚拟表格、类型推断和告警 | 基准大文件可查看，所有推断/错误规则有单元测试 |
| **P4** | Tab 管理、聚焦列、错误导航、命令面板、文件内/全局搜索、两类行号跳转 | 鼠标和键盘导航闭环，搜索可取消且定位准确 |
| **P5** | 单元格编辑、JSON 展示/编辑、文本视图、双视图同步、撤销重做、冲突检测和原子保存 | 表格/文本均可编辑；异常退出路径不静默丢数据 |
| **P6** | 快捷键补全、状态栏、视觉打磨、双平台性能验收、Windows x64/macOS arm64 打包、npm 三包发布工作流 | 通过功能回归、第 8 节基准和第 9.3 节安装验收，发布 npm 正式版本 |

每个阶段必须同时提交对应的自动化测试；涉及原生文件对话框、快捷键和窗口关闭确认的流程使用人工验收清单补充。
